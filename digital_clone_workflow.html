Manuale Operativo Digital Clone

Guida alla configurazione, gestione e sincronizzazione del clone virtuale.

🚀 Introduzione

Il Digital Clone è un'istanza avanzata di intelligenza artificiale progettata per replicare l'identità, la biografia e i tratti caratteriali di una persona reale. Utilizza un'architettura ibrida che combina Large Language Models (LLM) con un sistema di memoria semantica a lungo termine e una matrice emozionale dinamica.

Questa guida spiega come configurare i file core per personalizzare il comportamento del clone e come gestire il flusso di dati tra il browser e il server.

👤 Configurazione Identità

Il file principale per definire chi è il Digital Clone è persona_details.json. Questo file JSON contiene i dati anagrafici, i contatti e, soprattutto, il Background.

Struttura di persona_details.json

{
  "Persona": [
    {
      "Name": "Mario",
      "Surname": "Rossi",
      "BirthDate": "1980-01-01",
      "Background": "Nato nel 1980 a Roma...",
      "Website_01": "https://...",
      "Email": "info@example.com"
    }
  ]
}

Il campo Background è fondamentale: viene iniettato nel sistema di prompt per fornire al clone la consapevolezza di sé. Modifica questo campo per descrivere la tua storia, le tue competenze e la tua visione del mondo.

🎭 Matrice Emozionale

La personalità del clone non è statica. Attraverso il file emotional_matrix.txt, puoi calibrare i tratti caratteriali su una scala da 0 a 10. 

Empatia: 5/10
Humor: 9/10
Cinismo: 8/10
Curiosità: 9/10
Affidabilità: 10/10

Questi valori influenzano direttamente il System Message inviato alle API di OpenAI. Ad esempio, un alto valore di Humor renderà le risposte più scherzose, mentre un alto Cinismo porterà a risposte più scettiche e dirette.

📚 Biografia e Indicizzazione

La biografia estesa è gestita in modo modulare. Invece di un singolo file enorme, puoi suddividere la tua storia in più file di testo.

Gestione dei file

  1. Inserisci i file .txt nella cartella /bio/.
  2. Ogni file dovrebbe trattare un argomento specifico (es. istruzione.txt, carriera.txt, hobby.txt).

Indicizzazione Semantica

Affinché il chatbot possa "leggere" questi file, devono essere trasformati in vettori matematici (embeddings).

  • Clicca sul pulsante 🗂 Index Biography nella barra laterale del chatbot.
  • Verrà richiamato lo script indexBio.php che genererà il file biography_index.json.
  • Questo processo consuma token OpenAI (text-embedding-3-small) ma viene eseguito una sola volta per ogni aggiornamento della biografia.

🧠 Sistema di Memoria

Il Digital Clone utilizza due livelli di memoria per simulare la ritenzione cognitiva:

1. Memoria di Sessione

Contiene la cronologia recente della conversazione corrente. È volatile e viene persa al rinfresco della pagina, a meno che non venga esportata manualmente con 💾 Export this Session.

2. Memoria a Lungo Termine

Utilizza la tecnologia dei vettori (embeddings) per memorizzare informazioni importanti in modo permanente.

  • Salvataggio: Sotto ogni risposta del bot è presente il comando 💾 Save to Memory.
  • Recupero: Quando fai una domanda, il sistema cerca nelle memorie salvate quelle più rilevanti usando uno score composito: 70% Similitudine Semantica + 30% Recenza.
  • Persistenza: Inizialmente le memorie sono salvate nel localStorage del browser.

🔄 Sincronizzazione Browser-Server

Per evitare di perdere le memorie se pulisci la cache del browser o se cambi dispositivo, il sistema offre una sincronizzazione server-side.

Il Workflow di Sincronizzazione

Browser (localStorage)
▼
Pulsante "Sync Server Memory"
▼
saveMemory.php
▼
memoria_salvata.json (Server)

Quando clicchi 🔄 Sync Server Memory:

  1. Il browser invia tutte le memorie locali al server.
  2. Il server le unisce a quelle già presenti in memoria_salvata.json (evitando duplicati).
  3. Il browser scarica eventuali memorie presenti sul server che non ha ancora in locale.

In questo modo, il file memoria_salvata.json funge da "Cervello Centrale" condiviso.
