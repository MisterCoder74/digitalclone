Manuale Operativo Digital Clone

Guida completa alla configurazione, gestione e sincronizzazione del clone virtuale.

👤 1. Configurazione Identità

L'identità del clone è definita nel file persona_details.json. Questo file contiene le informazioni fondamentali che il chatbot utilizzerà per capire chi è e come rispondere.

File: persona_details.json

{
  "Persona": [
    {
      "Name": "Mario",
      "Surname": "Rossi",
      "BirthDate": "...",
      "Background": "Esempio di biografia breve che definisce l'identità...",
      "Website_01": "...",
      "Email": "..."
    }
  ]
}

Il campo Background è il più critico: descrive la storia personale, le competenze e la visione del mondo che il clone deve adottare come proprie.

📚 2. Gestione Biografia

Per fornire al clone una conoscenza approfondita del proprio passato e dei propri interessi, si utilizza un sistema di biografia modulare basato su file di testo.

Procedura:

  1. Inserisci i tuoi file biografici in formato .txt nella cartella /bio/.
  2. Puoi suddividere la biografia in più file per argomento (es. carriera.txt, interessi.txt, viaggi.txt).
  3. Per rendere questi file consultabili dal chatbot, è necessario indicizzarli cliccando sul pulsante 🗂 Index Biography nella barra laterale.

L'indicizzazione crea il file biography_index.json contenente i vettori (embeddings) necessari per la ricerca semantica durante la conversazione.

🎭 3. Matrice Emozionale

Il temperamento del clone può essere finemente calibrato tramite il file emotional_matrix.txt. Ogni tratto caratteriale è espresso su una scala da 0 a 10. 

Empatia: 7/10
Humor: 6/10
Cinismo: 2/10
Curiosità: 9/10
Affidabilità: 10/10

Questi parametri influenzano il tono delle risposte: un valore elevato di Humor renderà il clone più propenso alle battute, mentre un alto valore di Cinismo lo renderà più scettico e diretto.

🧠 4. Addestramento Manuale

Oltre alla biografia statica, il clone può imparare dalle interazioni in tempo reale.

  • Sotto ogni risposta del chatbot è presente il pulsante 💾 Save to Memory.
  • Cliccando su questo pulsante, l'informazione contenuta nella risposta viene salvata nella memoria a lungo termine del clone.
  • Questo permette di "istruire" il clone su nuovi fatti o opinioni emersi durante la chat.

🔄 5. Gestione Memoria e Sincronizzazione

Il sistema di memoria del Digital Clone è progettato per essere sia veloce che persistente, ma richiede attenzione nella gestione tra diversi dispositivi.

Memoria Locale vs Server

  • localStorage: Le memorie salvate risiedono inizialmente nella memoria del browser (localStorage). Questo significa che sono specifiche per il singolo browser e dispositivo in uso.
  • Problema delle Memorie Divergenti: Se utilizzi il clone su Chrome (PC) e poi su Safari (iPhone), i due dispositivi avranno memorie diverse. Questo crea una divergenza nella personalità e nei ricordi del clone.

⚠️ Procedura Obbligatoria di Riallineamento

Per sincronizzare tutti i dispositivi con la "Verità Centrale" ospitata sul server, segui rigorosamente questa procedura:

1. Clicca su
"Clear Long Memory"
(Pulisce la memoria locale divergente)
▼
2. Clicca subito dopo su
"Sync Server Memory"
(Scarica la memoria consolidata dal server)

Questa operazione garantisce che il browser corrente sia perfettamente allineato con l'archivio centrale (memoria_salvata.json) presente sul server.
