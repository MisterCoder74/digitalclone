1. DESCRIZIONE DI MASSIMA DEL PROGETTO

1.1 Cos'è

Un'applicazione web interattiva che simula un "clone digitale" di Alessandro Demontis, basato su modelli GPT di OpenAI. È un chatbot avanzato con memoria a lungo termine, analisi di documenti, analisi di immagini e ricerca web integrata.

1.2 Tecnologie Utilizzate

Frontend: HTML5, CSS3 (glassmorphism e gradient), JavaScript vanilla + jQuery

Backend API: OpenAI API (GPT-4.1-mini, GPT-5-search-api, text-embedding-3-small)

Storage: localStorage del browser (per API key e memoria)

Architettura: Single Page Application (SPA) con interfaccia moderna

1.3 Descrizione Interfaccia

L'interfaccia è divisa in due colonne principali:

SIDEBAR (260px): Contiene avatar del clone, gestione API key, analizzatore documenti, analizzatore immagini, gestione memoria, ricerca chat.

CHAT AREA (flex): Cronologia messaggi con bubble style (blu per utente, viola per bot), area input con textarea e bottoni Send/Search Web.

Stile: Glassmorphism moderno con gradiente di sfondo (blu scuro → viola → nero), blur effect, trasparenza, icone emoji, responsive design.

2. REPORT DETTAGLIATA DELLE FUNZIONALITÀ E WORKFLOW

2.1 Gestione API Key

Storage: localStorage.getItem("openaikey") / localStorage.setItem("openaikey", apiKey)

Funzionalità: Salva/carica API key di OpenAI in localStorage

Workflow:

1. Utente clicca "🔑 API Key"

2. Modal si apre con input field

3. API key salvata in localStorage

4. Visualizzata (mascherata) nel campo password

2.2 Chat Principale - sendMessage()

Workflow: User Input → sendMessage() → getChatGPTResponse() → openAIChat() → searchLongMemory() → OpenAI API Call → saveToLongMemory() → showMessage() → Display

Model: gpt-4.1-mini

Max tokens: 15,999

Token pricing: Prompt €0.00000125, Response €0.00001

Memoria: Ricerca memoria a lungo termine prima della richiesta

Output: Messaggio con header timestamp, contenuto, token info, bottone copy

2.3 Ricerca Web - sendWeb()

Workflow: User Input → sendWeb() → getChatGPTResponseWeb() → openAIChat() → web_search_options: "medium" → OpenAI API (gpt-5-search-api) → Extract Citations → showMessageWeb() → Display con References

Model: gpt-5-search-api

Token pricing: Prompt €0.00000015, Response €0.0000006

Features: Ricerca web in tempo reale, estrazione automatica di citazioni/riferimenti, link cliccabili alle fonti, pulizia URL dalla risposta

2.4 Analisi Documenti - analyzeFile()

Tipi supportati:

OpenAI native: .pdf, .jpg, .png, .gif, .webp

Text-based: .txt, .xml, .html, .css, .js, .json, .vb, .php, .py

Workflow:

1. Upload file a OpenAI (max 20MB)

2. Ricevi file_id

3. Includi file_id nelle richieste chat successive

4. Analisi contestuale del documento

2.5 Analisi Immagini - getChatGPTResponseIMG()

Workflow: Image Upload → vupload.php → Get Image URL → openAIChat() with vision → Extract image analysis → showMessage() with image preview

Features: Upload tramite form multipart, preview dell'immagine nel chat, analisi con vision capabilities

2.6 Memoria a Lungo Termine

2.6.1 Salvataggio - saveToLongMemory()

Processo:

1. Combina user input + bot response

2. Genera embedding via OpenAI (text-embedding-3-small)

3. Salva in localStorage con timestamp

Configurazione MEMORY_CONFIG:

MAX_MEMORIES: 1500 (max memories da mantenere)
MAX_DAYS: 365 (retention period)
SIMILARITY_THRESHOLD: 0.5 (min score per inclusione)
SIMILARITY_WEIGHT: 0.7 (peso similarità semantica)
RECENCY_WEIGHT: 0.3 (peso recency temporale)
ENABLE_COMPOSITE_SCORING: true
2.6.2 Ricerca - searchLongMemory()

Algoritmo:

1. Query → Generate Embedding

2. For each Memory: Calculate Cosine Similarity + Recency Score

3. Composite Score = (similarity × 0.7) + (recency × 0.3)

4. Filter by SIMILARITY_THRESHOLD (0.5)

5. Sort by Composite Score (descending)

6. Return Top-K (default: 3)

Formula Recency Score: e^(-k × age), dove k = -ln(0.01) / maxAgeMs

Assicura: 1.0 (oggi) → 0.01 (MAX_DAYS)

2.7 Gestione Memoria - Memory Management UI

Show Memory: Apre modal con cronologia chat completa

Clear Long Memory: Cancella tutti i memories salvati

Export Long Memory: Scarica memories in JSON

Export this Session: Scarica chat corrente in TXT

Memory Status: Mostra numero memories salvati e limiti

2.8 Ricerca Chat - searchChat()

Workflow: Search Term → Filter message-rows → Match case-insensitive → Show/Hide based on match → Clear search input

3.2 Principi Gestione Memoria a Breve Termine

chatMemory Array: Array JavaScript che mantiene la cronologia della conversazione

Struttura:

[
  { role: "user", content: "user message" },
  { role: "assistant", content: "bot response" },
  ...
]

Lifecycle:

• Inizializzazione: chatMemory = [] (vuoto all'inizio)

• Accumulo: Ogni scambio aggiunge 2 elementi (user + assistant)

• Passaggio API: Intero array inviato a OpenAI per context

• Reset: chatMemory = [] quando si carica un nuovo documento

• Limite: Max 15,999 tokens per richiesta (OpenAI limit)

✓ Vantaggi:

• Context completo della conversazione

• Coerenza nelle risposte

• Memoria conversazionale immediata

✗ Svantaggi:

• Cresce linearmente con conversazione

• Costo token aumenta

• Perso al refresh pagina

3.3 Principi Gestione Memoria a Lungo Termine

Struttura dati localStorage["vivacityLongMemory"]:

[
  {
    testo: "Utente: ...\\nBot: ...",
    vettore: [0.123, -0.456, ...],  // 1536 dimensioni
    data: "2025-04-11T10:30:00.000Z"
  },
  ...
]

Algoritmo di Scoring Composito (Phase 4):

1. SIMILARITY SCORE (Semantic): Cosine similarity tra query embedding e memory (range [0,1])

2. RECENCY SCORE (Temporal): Exponential decay basato su age (range [0,1])

3. COMPOSITE SCORE: (similarity × 0.7) + (recency × 0.3)

4. FILTERING & RANKING: Filter by threshold (0.5), sort descending, return top-3

Formula Recency Score: e^(-k × age), dove k = -ln(0.01) / MAX_DAYS_MS

Assicura: 1.0 (oggi) → 0.01 (MAX_DAYS = 365 giorni)

✓ Vantaggi:

• Memoria persistente tra sessioni

• Ricerca semantica (non keyword-based)

• Bilanciamento tra relevanza e recency

• Configurabile a runtime

• Retrocompatibile (può disabilitare composite scoring)

✗ Svantaggi:

• Storage limitato (localStorage ~5-10MB)

• Costo embedding per ogni salvataggio

• Costo embedding per ogni ricerca

• Latenza ricerca aumenta con numero memories
